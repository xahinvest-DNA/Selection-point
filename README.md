# Selection Point

**Selection Point** — самостоятельная система восстановления способности выбора.

## Текущее состояние

- Активная фаза: **Фаза 3 — точная архитектура пяти ступеней**.
- Активная ступень: **Ступень 1 «Жизнь внутри автоматизма»**.
- Утверждены параметры SP-S1-P01–SP-S1-P11:
  1. состояние человека;
  2. способ восприятия реальности;
  3. способ восприятия себя;
  4. что управляет решениями;
  5. доступная точка выбора;
  6. главная иллюзия — «Иллюзия неизбежного продолжения»;
  7. главная боль — «Боль повторяющегося расхождения между намерением и траекторией»;
  8. следующий навык — «Реконструкция конкретного автоматического эпизода»;
  9. телесные проявления — «Тело внутри автоматического контура»;
  10. жизненные проявления — «Текущая жизнь как позиция траектории»;
  11. соответствующие практики — функциональный профиль «От автоматического эпизода к доступному участию».
- **SP-S1-P12 «Критерии перехода» не открыт.**

Актуальная контрольная точка:

- `docs/FOUNDATION/CURRENT_PROJECT_STATE.md`;
- `docs/CURRENT_STATE.md` — короткая совместимая точка входа.

## Последнее утверждённое положение

**SP-S1-P11 — «Соответствующие практики».**

> **Соответствующие практики первой ступени должны помогать человеку локализовать один конкретный автоматический эпизод, реконструировать его как последовательность условий, состояния, воспринятой ситуации, активированного образа себя, оперативной задачи, привычного хода, немедленного результата, отложенной цены и созданной следующей позиции. Практическая работа должна завершаться обнаружением первого реально доступного участка участия и одним выполнимым изменением продолжения, последствий или условий следующего цикла. Она не требует предотвращения первой реакции, полного понимания повторяющегося сценария или устойчивого сохранения выбора под напряжением. Её задача — создать первый наблюдаемый опыт того, что автоматический эпизод можно различить, а его дальнейшая траектория не обязана полностью определяться уже возникшей реакцией.**

Короткая формула:

> **Не остановить автоматизм идеально, а сделать один эпизод различимым и изменить хотя бы одно его продолжение.**

Полный источник:

Канонический источник:

- `docs/FOUNDATION/CANONICAL/03_STAGE_1_P11_CANONICAL.md`.

## Граница практического параметра

В Фазе 3 параметр 11 определяет функциональный профиль будущих практик, но не готовые упражнения.

Конкретные инструкции, длительность, частота, рабочие листы, сопровождение и интерфейс проектируются только на последующих фазах. Полная конструкция практик относится к Фазе 5.

Источник:

- `docs/FOUNDATION/PRACTICE_PARAMETER_BOUNDARY.md`.

## Сквозная визуальная модель

Утверждена авторская модель Андрея **SP-VM-01 «Граф траекторий выбора»**:

```text
текущая позиция
→ поле доступных ходов
→ следующий ход
→ новая позиция
→ новое поле ходов
```

Модель связана с SP-S1-P05–SP-S1-P11.

Для SP-S1-P11 она поддерживает связь:

```text
исходная позиция
→ совершённый ход
→ созданная следующая позиция
→ доступное изменение продолжения
```

## Что сейчас нельзя делать

- не открывать SP-S1-P12 без отдельного указания Андрея;
- не превращать функциональные классы SP-S1-P11 в готовые упражнения;
- не определять критерии перехода автоматически из минимального результата одного применения;
- не создавать инструкции, рабочие листы, длительность и частоту практик;
- не превращать граф траекторий в готовую практику;
- не проектировать карту курса, пилот, приложение или технический стек;
- не требовать от первой ступени способностей следующих ступеней.

> Не скорость генерации, а последовательность утверждённых решений создаёт архитектуру Selection Point.
