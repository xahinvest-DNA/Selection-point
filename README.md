# Selection Point

**Selection Point** — самостоятельная система восстановления способности выбора.

## Текущее состояние

- Активная фаза: **Фаза 3 — точная архитектура пяти ступеней**.
- Активная ступень: **Ступень 1 «Жизнь внутри автоматизма»**.
- Утверждены параметры SP-S1-P01–SP-S1-P07:
  1. состояние человека;
  2. способ восприятия реальности;
  3. способ восприятия себя;
  4. что управляет решениями;
  5. доступная точка выбора;
  6. главная иллюзия — «Иллюзия неизбежного продолжения»;
  7. главная боль — «Боль повторяющегося расхождения между намерением и траекторией».
- **SP-S1-P08 не открыт** и не рассматривается до отдельного указания Андрея.

Актуальная контрольная точка:

- `docs/FOUNDATION/CURRENT_PROJECT_STATE.md`;
- `docs/CURRENT_STATE.md` — короткая совместимая точка входа.

## Последнее утверждённое положение

> **Главная боль первой ступени состоит в повторяющемся расхождении между значимым для человека направлением и траекторией, которую создают его фактические действия.**

Короткая формула:

> **Каждый шаг казался необходимым — но снова привёл меня туда, куда я не хотел.**

Полный источник:

- `docs/FOUNDATION/FIVE_STAGES_THEORY/07_STAGE_1_MAIN_PAIN.md`.

Канонический источник:

- `docs/FOUNDATION/CANONICAL/02_STAGE_1_CANONICAL.md`.

## Сквозная визуальная модель

Утверждена авторская модель Андрея **SP-VM-01 «Граф траекторий выбора»**:

```text
текущая позиция
→ поле доступных ходов
→ следующий ход
→ новая позиция
→ новое поле ходов
```

Модель связана с SP-S1-P05–SP-S1-P07 и показывает позицию, ход, новую позицию, автоматический контур, новую ветвь, переживаемую неизбежность и расхождение фактической траектории со значимым направлением.

- `docs/FOUNDATION/VISUAL_MODELS/assets/choice_trajectory_graph.svg`.

## Что сейчас нельзя делать

- не открывать SP-S1-P08 без отдельного указания Андрея;
- не проектировать следующий навык;
- не утверждать практики;
- не определять критерии перехода;
- не формулировать ловушку ступени;
- не проектировать курс, модули, длительность, приложение или технический стек.

> Не скорость генерации, а последовательность утверждённых решений создаёт архитектуру Selection Point.
