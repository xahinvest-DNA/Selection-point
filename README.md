# Selection Point

**Selection Point** — самостоятельная система восстановления способности выбора.

## Текущее состояние

- Активная фаза: **Фаза 3 — точная архитектура пяти ступеней**.
- **Ступень 1 «Жизнь внутри автоматизма» утверждена полностью.**
- Утверждены параметры SP-S1-P01–SP-S1-P13.
- **Ступень 2 «Обнаружение повторяющегося сценария» не открыта.**

Актуальная контрольная точка:

- `docs/FOUNDATION/CURRENT_PROJECT_STATE.md`;
- `docs/CURRENT_STATE.md` — короткая совместимая точка входа.

## Последнее утверждённое положение

**SP-S1-P13 — «Ловушка первой ступени».**

Рабочее название:

> **«Ловушка объяснённого автоматизма».**

Утверждённая формула:

> **Я понял, как это происходит, — и снова заменил выбор пониманием.**

Дополнительная формула риска превосходства:

> **Я вижу автоматизм других, но не превращаю своё видение в участие в собственной жизни.**

Ловушка первой ступени возникает, когда способность реконструировать автоматический эпизод превращается в новый автоматический контур: объяснение, правильный язык, контроль, самообвинение, диагностика других или превосходство наблюдателя заменяют доступное участие и изменение продолжения.

Полный источник:

- `docs/FOUNDATION/FIVE_STAGES_THEORY/13_STAGE_1_STAGE_TRAP.md`.

Канонический источник:

- `docs/FOUNDATION/CANONICAL/05_STAGE_1_P13_CANONICAL.md`.

## Новые риски

- `SP-RISK-026` — объяснение как новый автоматический контур;
- `SP-RISK-027` — превосходство наблюдателя при личной неинтегрированности.

## Сквозная визуальная модель

SP-VM-01 «Граф траекторий выбора» связана с SP-S1-P05–SP-S1-P13.

Для P13 объяснение, диагностика других или чувство превосходства могут быть представлены как ход:

```text
напряжение от повторения
→ объяснение / диагностика / чувство превосходства
→ временная ясность или контроль
→ отсутствие собственного доступного участия
→ следующая позиция почти не изменилась
```

## Что сейчас нельзя делать

- не открывать Ступень 2 без отдельного указания Андрея;
- не превращать P13 в практику выхода из ловушки раньше Фазы 5;
- не проектировать курс, пилот, приложение или технический стек;
- не превращать способность видеть других в статус или доказательство интеграции;
- не считать объяснение автоматизма достаточным изменением;
- не использовать P13 для самообвинения;
- не объявлять модель научно доказанной без внешней проверки.

> Не скорость генерации, а последовательность утверждённых решений создаёт архитектуру Selection Point.
