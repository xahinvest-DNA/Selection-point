# Шаблон канонической карточки параметра

Используется после явного утверждения каждого параметра пяти ступеней.

```markdown
# <ID> — <Название параметра>

**Статус:** утверждено  
**Ступень:** <номер и рабочее название>  
**Параметр:** <номер из 13>  
**Дата утверждения:** <дата>  
**Полный источник:** `<путь к теоретическому файлу>`

## Центральный тезис

<Одно точное утверждение, определяющее параметр.>

## Что добавляет параметр

<Какой новый элемент архитектуры появляется и чем он отличается от уже утверждённых параметров.>

## Механизм

<Краткая последовательность или объяснение механизма.>

```text
<схема при необходимости>
```

## Наблюдаемые признаки

- <признак 1>;
- <признак 2>;
- <признак 3>.

## Главные различения

- <что нельзя смешивать>;
- <что параметр не означает>;
- <какая соседняя идея остаётся отдельной>.

## Границы и ограничения

<Методологические, психологические, медицинские, этические и практические ограничения.>

- <ID следующего положения>.

- <что нельзя проектировать иначе>.

## Компактная утверждённая формулировка

## Короткая формула

> <Одна короткая формула.>
```

## Контроль перед сохранением

1. Смысл карточки полностью соответствует утверждённому теоретическому файлу.
2. В карточку не добавлено ничего, что не было утверждено.
3. Полный источник указан и существует.
4. Предыдущие параметры не пересказаны повторно без необходимости.
5. Общие термины, примеры и риски вынесены в библиотеки или связаны ID.
6. Соседние параметры не смешаны.
7. Карточка достаточна для проектирования следующего слоя.
8. Карточка не используется как замена полного теоретического основания.
