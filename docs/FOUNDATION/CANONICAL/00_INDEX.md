# Канонический слой Selection Point

**Статус:** действующий производный слой утверждённой архитектуры  
**Дата создания:** 4 июля 2026 года  
**Дата обновления:** 6 июля 2026 года

Канонический слой хранит компактную, структурированную форму уже утверждённых положений. Он создаётся из полного теоретического слоя и используется для навигации, проверки целостности, проектирования следующих фаз и последующей сборки материалов доставки.

## Главное различие

```text
полный теоретический слой
= сохраняет весь разбор, аргументацию, примеры, исключения и путь к решению

канонический слой
= фиксирует утверждённую архитектуру в компактной и сопоставимой форме

библиотеки
= хранят повторно используемые термины, тезисы, примеры, риски и связи

визуальные модели
= показывают связи и динамику нескольких утверждённых положений

слой доставки
= курс, практика, рабочая тетрадь, пилот, интерфейс и другие формы для участника
```

Полные теоретические документы не сокращаются и не заменяются каноническими карточками.

## Состав

1. `00_CANONICAL_RULES.md` — правила четырёхслойной архитектуры, правило дельты и порядок обновления.
2. `01_HUMAN_CHANGE_MODEL_CANONICAL.md` — каноническая карта восьми положений модели человека и изменения.
3. `02_STAGE_1_CANONICAL.md` — канонические карточки утверждённых параметров первой ступени.
4. `PARAMETER_CARD_TEMPLATE.md` — обязательный шаблон для всех следующих параметров пяти ступеней.
5. `DELIVERY_LAYER_RULES.md` — правила будущей сборки курса, практик, пилота и цифрового продукта.

## Правило приоритета

Если каноническая карточка расходится с полным утверждённым теоретическим документом или с явным решением Андрея, приоритет имеет полное утверждённое положение. Канонический слой должен быть исправлен без изменения смысла исходного решения.

Канонический слой и визуальная модель нельзя использовать для незаметного сокращения, расширения или переосмысления утверждённой архитектуры.
