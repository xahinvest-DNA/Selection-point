# Дополнение к контрольной точке операционного протокола

**Статус:** действующее дополнение к `PROJECT_OPERATING_PROTOCOL.md`  
**Дата:** 6 июля 2026 года  
**Основание:** явное утверждение Андреем SP-S1-P11

Это дополнение обновляет только текущую контрольную точку операционного протокола. Общие правила, роли, иерархия источников, порядок фаз и рабочий цикл остаются без изменений.

## Актуальная контрольная точка

- Активная фаза: Фаза 3 — точная архитектура пяти ступеней.
- Активная ступень: Ступень 1 — «Жизнь внутри автоматизма».
- SP-S1-P01–SP-S1-P11 утверждены.
- Последнее утверждённое положение: SP-S1-P11 «Соответствующие практики».
- SP-S1-P12 «Критерии перехода» не открыт.

## Утверждённая короткая формула SP-S1-P11

> **Не остановить автоматизм идеально, а сделать один эпизод различимым и изменить хотя бы одно его продолжение.**

## Источники

- полный: `FIVE_STAGES_THEORY/11_STAGE_1_CORRESPONDING_PRACTICES.md`;
- канонический: `CANONICAL/03_STAGE_1_P11_CANONICAL.md`;
- граница параметра и Фазы 5: `PRACTICE_PARAMETER_BOUNDARY.md`;
- актуальный статус: `CURRENT_PROJECT_STATE.md`.

## Статус продолжения

> **Работа остановлена после полной фиксации SP-S1-P11. SP-S1-P12 не открывается до отдельного указания Андрея.**

При расхождении старой контрольной точки внутри `PROJECT_OPERATING_PROTOCOL.md` с этим дополнением и `CURRENT_PROJECT_STATE.md` применяются последнее явное решение Андрея и актуальный динамический документ.
