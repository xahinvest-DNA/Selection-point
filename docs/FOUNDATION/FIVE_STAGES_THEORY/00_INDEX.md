# Теоретический слой точной архитектуры пяти ступеней

**Статус:** активная разработка  
**Фаза:** 3 — точная архитектура пяти ступеней  
**Дата обновления:** 4 июля 2026 года

Эта папка хранит полный утверждённый теоретический материал по тринадцати параметрам каждой из пяти ступеней Selection Point.

Рабочие названия ступеней остаются предварительными до отдельного утверждения их точной архитектуры.

## Связанные слои

Полный теоретический слой используется совместно с:

- `../CANONICAL/00_CANONICAL_RULES.md` — обязательные правила канонизации и правило дельты;
- `../CANONICAL/02_STAGE_1_CANONICAL.md` — компактная действующая архитектура первой ступени;
- `../LIBRARIES/00_INDEX.md` — термины, утверждения, примеры, риски, доказательный статус и связи.

Полный теоретический файл сохраняет весь разбор. Каноническая карточка сохраняет компактную рабочую архитектуру. Ни один слой не заменяет другой.

# Порядок разработки каждой ступени

Для каждой ступени последовательно утверждаются:

1. состояние человека;
2. способ восприятия реальности;
3. способ восприятия себя;
4. что управляет решениями;
5. доступная точка выбора;
6. главная иллюзия;
7. главная боль;
8. следующий навык;
9. телесные проявления;
10. жизненные проявления;
11. соответствующие практики;
12. критерии перехода;
13. ловушка ступени.

Каждый параметр обсуждается, утверждается и сохраняется отдельно. Утверждение одного параметра не означает автоматического утверждения остальных.

## Обязательный результат после утверждения параметра

После явного утверждения создаются или обновляются одновременно:

1. полный теоретический файл в этой папке;
2. каноническая карточка ступени в `../CANONICAL/`;
3. запись в `../LIBRARIES/CLAIMS_REGISTRY.md`;
4. новые термины, если они появились;
5. повторно используемые примеры;
6. общие риски и границы;
7. карта перекрёстных связей;
8. доказательный статус;
9. индекс этой папки;
10. текущая контрольная точка.

Только после этого открывается следующий параметр.

## Правило дельты для следующего полного файла

Новый теоретический файл обязан:

- перечислить утверждённые ID, на которые он опирается;
- обозначить один новый вопрос;
- явно назвать соседние параметры, которые не рассматриваются;
- не повторять полный разбор уже утверждённых параметров;
- ссылаться на общие термины, примеры и риски из библиотек;
- подробно раскрывать только новый смысл, исключения и специфические ограничения.
