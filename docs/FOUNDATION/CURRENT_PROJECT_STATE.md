# 3. Архитектура управления теоретическим объёмом

Действует система:

```text
полная теория
→ каноническая архитектура
→ общие библиотеки
→ визуальные модели
→ материалы доставки
```

## Полная теория

Сохраняет весь разбор, аргументацию, примеры, исключения и ограничения.

## Канонический слой

Расположение: `docs/FOUNDATION/CANONICAL/`.

Он фиксирует компактную утверждённую архитектуру и является основным входом для следующих фаз.

## Общие библиотеки

Расположение: `docs/FOUNDATION/LIBRARIES/`.

Содержат:

- термины;
- утверждения;
- примеры;
- риски и границы;
- доказательный статус;
- перекрёстные связи.

## Визуальные модели

Расположение: `docs/FOUNDATION/VISUAL_MODELS/`.

Визуальная модель не заменяет полный источник, не является доказательством и обязана содержать границы метафоры.

## Правило дельты

Каждый следующий полный теоретический файл подробно раскрывает только новый архитектурный вопрос. Утверждённые механизмы кратко обозначаются стабильными ID и не пересказываются повторно без необходимости.

## Обязательный цикл после утверждения

```text
полный теоретический файл
→ каноническая карточка
→ реестр утверждений
→ термины

→ риски
→ доказательный статус
```

# 5. Текущий прогресс ступени 1

## SP-S1-P01 — Состояние человека

**Статус: утверждено.**

Человек находится внутри автоматической реакции и недостаточно различает её звенья. Движение начинается из личного расхождения между желаемым и повторяемым; первым ходом может быть внимание к одному эпизоду.

Полный источник: `FIVE_STAGES_THEORY/01_STAGE_1_STATE.md`.

## SP-S1-P02 — Способ восприятия реальности

**Статус: утверждено.**

Человек смотрит не на текущую модель реальности, а из неё. Первый разрыв возникает между «так всё и есть» и «так я сейчас это воспринимаю».

Полный источник: `FIVE_STAGES_THEORY/02_STAGE_1_REALITY_PERCEPTION.md`.

## SP-S1-P03 — Способ восприятия себя

**Статус: утверждено.**

Человек принимает активированный образ себя за сущность, а знакомую роль — за границу допустимых действий. Первый разрыв локализует роль в конкретном эпизоде.

Полный источник: `FIVE_STAGES_THEORY/03_STAGE_1_SELF_PERCEPTION.md`.

## SP-S1-P04 — Что управляет решениями

**Статус: утверждено.**

Решением преимущественно управляет доминирующая оперативная задача текущего эпизода. Приоритет получает знакомое и психологически доступное средство, обещающее нужный немедленный результат.

Полный источник: `FIVE_STAGES_THEORY/04_STAGE_1_DECISION_DRIVERS.md`.

## SP-S1-P05 — Доступная точка выбора

**Статус: утверждено 5 июля 2026 года.**

Доступная точка выбора — первый различимый участок автоматической цепочки, на котором человеку доступен хотя бы один реально выполнимый ход, изменяющий дальнейшее продолжение.

Точка может находиться:

- до основного действия;
- во время уже начавшегося действия;
- после действия внутри продолжающейся цепочки;
- при подготовке следующего цикла.

Доступность определяется пересечением:

1. различимости;
2. выполнимости;
3. влияния на дальнейшую траекторию.

Короткая формула:

> Доступная точка выбора — первый замеченный участок цепочки, где следующий ход ещё можно изменить.

Полный источник:

- `FIVE_STAGES_THEORY/05_STAGE_1_AVAILABLE_POINT_OF_CHOICE.md`.

Канонический источник:

- `CANONICAL/02_STAGE_1_CANONICAL.md`.

# 6. Утверждённая сквозная визуальная модель SP-VM-01

**Название:** «Граф траекторий выбора»  
**Автор исходной идеи и наброска:** Андрей  
**Статус:** утверждено 5 июля 2026 года

```text
текущая позиция
→ поле доступных ходов
→ следующий ход
→ новая позиция
→ новое поле ходов
```

Модель показывает:

- локальную точку выбора;
- последовательность позиций и ходов;
- повторяющийся автоматический контур;
- новую ветвь траектории;
- изменение поля будущих возможностей после каждого действия.

Главные формулы:

> Каждый ход создаёт поле следующего хода.

> Мы не выбираем всю жизнь сразу. Мы совершаем следующий ход и этим создаём позицию, из которой будет сделан следующий выбор.

Модель не означает равной доступности, безопасности или обратимости всех направлений.

Источники:

- `VISUAL_MODELS/01_CHOICE_TRAJECTORY_GRAPH.md`;
- `VISUAL_MODELS/assets/choice_trajectory_graph.svg`.

# 8. Следующий вопрос

> **SP-S1-P06 — главная иллюзия первой ступени.**

Необходимо определить одно центральное заблуждение, которое удерживает целостную логику первой ступени и делает автоматическое продолжение естественным или неизбежным.

Нельзя автоматически объявить главной иллюзией одну из отдельных формул:

- «такова реальность»;
- «я такой»;
- «иначе нельзя»;
- «это единственный способ решить срочную задачу»;
- «после первого действия уже всё равно».

Нужно найти центральное заблуждение, которое объединяет SP-S1-P01–SP-S1-P05, не стирая различий между ними.

Нельзя одновременно утверждать:

- главную боль;
- следующий навык;
- практику;
- критерии перехода;
- ловушку ступени;
- параметры следующих ступеней.
