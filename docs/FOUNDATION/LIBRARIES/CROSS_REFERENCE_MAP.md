# Карта перекрёстных связей Selection Point

**Статус:** действующая навигационная библиотека  
**Дата обновления:** 7 июля 2026 года

Карта показывает, какие утверждения являются основанием для следующих элементов и где искать полный смысл.

## Связи модели человека со ступенью 1

| Положение | Основное влияние на ступень 1 |
|---|---|
| SP-HCM-01 — воспринятая модель | SP-S1-P02; участвует в реконструкции SP-S1-P08, функциональном цикле SP-S1-P11 и различении факта и гипотезы SP-S1-P12 |
| SP-HCM-02 — состояние | SP-S1-P01–SP-S1-P12; влияет на доступность действия, реконструкцию и реалистичность критерия перехода |
| SP-HCM-03 — автоматическая цепочка | SP-S1-P01, SP-S1-P05–SP-S1-P12; задаёт структуру реконструируемого и сопоставимого эпизода |
| SP-HCM-04 — образ себя и сценарий | SP-S1-P03–SP-S1-P12; образ себя должен оставаться гипотезой конкретного эпизода, а не превращаться в преждевременный сценарий |
| SP-HCM-05 — действие меняет действующего | SP-S1-P05–SP-S1-P12 и SP-VM-01; действие связывается со следующей позицией и наблюдаемым влиянием |
| SP-HCM-06 — обратная связь | SP-S1-P05–SP-S1-P12; критерий перехода требует использования последствий, а не только правильного объяснения |
| SP-HCM-07 — множественные точки восстановления | SP-S1-P05–SP-S1-P12; доступное участие может появиться до, во время или после первого действия |
| SP-HCM-08 — повторяющиеся циклы | SP-S1-P06–SP-S1-P12 и SP-VM-01; повторяемость операции отличает разовое понимание от готовности к переходу |

## Внутренняя последовательность первой ступени

```text
SP-S1-P01
общая позиция человека внутри автоматизма

→ SP-S1-P02
какую ситуацию он переживает как реальность

→ SP-S1-P03
кем он переживает себя внутри этой ситуации

→ SP-S1-P04
какую срочную задачу должно решить действие

→ SP-S1-P05
где участие в этой цепочке реально доступно

→ SP-S1-P06
какое центральное заблуждение удерживает логику ступени

→ SP-S1-P07
какое повторяющееся расхождение составляет главную боль

→ SP-S1-P08
какая первая способность делает один эпизод различимым

→ SP-S1-P09
как тело участвует в автоматическом контуре

→ SP-S1-P10
как текущая жизненная позиция хранит след предыдущей траектории

→ SP-S1-P11
какие функции должна выполнять практика, чтобы реконструкция завершалась доступным участием

→ SP-S1-P12
по каким качественным признакам операция стала повторяемой, частично самостоятельной и готовой к будущему сопоставлению эпизодов
```

SP-S1-P13 не выводится автоматически из SP-S1-P12 и не открыт.

## Связь SP-S1-P12 с SP-VM-01

SP-VM-01 представляет каждый эпизод как отдельную последовательность:

```text
позиция
→ ход
→ следующая позиция
```

Несколько таких последовательностей могут быть сохранены как сопоставимые эпизоды. Способ их сравнения и обнаружения повторяющихся звеньев относится ко второй ступени и пока не утверждается.

Связанные элементы:

- полный источник: `FIVE_STAGES_THEORY/12_STAGE_1_TRANSITION_CRITERIA.md`;
- каноническая карточка: `CANONICAL/04_STAGE_1_P12_CANONICAL.md`;
- архитектурная граница: `TRANSITION_CRITERIA_BOUNDARY.md`;
- новый термин: SP-TERM-COMPARABLE-EPISODE;
- новые риски: SP-RISK-024, SP-RISK-025;
- доказательный статус: строка SP-S1-P12 в `EVIDENCE_MAP.md`.


## Правило обновления

После утверждения нового параметра необходимо:

1. добавить связь с его основаниями;
2. указать, что он подготавливает, только если следующий параметр действительно открыт;
3. связать новые термины либо зафиксировать, что новых терминов нет;
4. указать подходящие примеры;
5. зафиксировать применимые риски;
6. связать визуальную модель, если она добавлена или уточнена;
7. не создавать связь с ещё не утверждённым выводом как с фактом.
