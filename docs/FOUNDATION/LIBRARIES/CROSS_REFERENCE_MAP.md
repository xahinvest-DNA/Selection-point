# Карта перекрёстных связей Selection Point

**Статус:** действующая навигационная библиотека

Карта показывает, какие утверждения являются основанием для следующих элементов и где искать полный смысл.

## Связи модели человека со ступенью 1

| Положение | Основное влияние на ступень 1 |
|---|---|
| SP-HCM-01 — воспринятая модель | SP-S1-P02; участвует в SP-S1-P01 и SP-S1-P04 |
| SP-HCM-02 — состояние | SP-S1-P01, SP-S1-P02, SP-S1-P04 |
| SP-HCM-03 — автоматическая цепочка | SP-S1-P01; основание будущего SP-S1-P05 |
| SP-HCM-04 — образ себя и сценарий | SP-S1-P03, SP-S1-P04 |
| SP-HCM-05 — действие меняет действующего | будущие последствия, боль, критерии перехода и ловушка ступени |
| SP-HCM-06 — обратная связь | будущие практики и критерии перехода |
| SP-HCM-07 — множественные точки восстановления | непосредственное основание SP-S1-P05 |
| SP-HCM-08 — повторяющиеся циклы | будущие практики, критерии перехода и следующая ступень |

## Внутренняя последовательность первой ступени

```text
SP-S1-P01
общая позиция человека внутри автоматизма

→ SP-S1-P02
какую ситуацию он переживает как реальность

→ SP-S1-P03
кем он переживает себя внутри этой ситуации

→ SP-S1-P04
какую срочную задачу должно решить действие

→ SP-S1-P05
где участие в этой цепочке реально доступно
```

Следующие параметры нельзя выводить автоматически из первых пяти:

- SP-S1-P06 — центральная иллюзия;
- SP-S1-P07 — главная боль;
- SP-S1-P08 — следующий навык;
- SP-S1-P09 — телесные проявления;
- SP-S1-P10 — жизненные проявления;
- SP-S1-P11 — практики;
- SP-S1-P12 — критерии перехода;
- SP-S1-P13 — ловушка ступени.

Каждый из них требует отдельного обсуждения и утверждения.

## Навигация по слоям

```text
утверждение или параметр
→ CLAIMS_REGISTRY.md
→ каноническая карточка
→ полный теоретический источник
→ связанные термины, примеры, риски и доказательный статус
```

## Правило обновления

После утверждения нового параметра необходимо:

1. добавить связь с его основаниями;
2. указать, что он подготавливает;
3. связать новые термины;
4. указать подходящие примеры;
5. зафиксировать применимые риски;
6. не создавать связь с ещё не утверждённым выводом как с фактом.
