# PRODUCT_BRIEF

## Рабочее название

**Selection Point**

Дополнительное название протокола v1:

**Тело без войны**

## Продуктовая формула

Selection Point помогает человеку увидеть, где он действует автоматически, и заменить автоматическую реакцию точной внутренней задачей и ближайшим действием.

## Версия v1

Первая версия — мобильное приложение для 14-дневного протокола формирования здорового и энергичного тела.

Это не диета, не фитнес-трекер и не приложение подсчёта калорий в первую очередь.

Это система наблюдения, выбора и обратной связи.

## Основная проблема пользователя

Пользователь хочет похудеть, стать энергичнее и управляемее, но обычно действует через старую схему:

```
вдохновение → жёсткий контроль → усталость → срыв → вина → откат
```

Selection Point должен заменить эту схему на:

```
факт → наблюдение → главный механизм → внутренняя задача → маленький ход → обратная связь → новый цикл
```

## Целевая аудитория v1

Первая аудитория:

- люди с лишним весом или ощущением тяжёлого тела;
- люди с низкой энергией;
- люди, которые много раз начинали диеты и бросали;
- люди, у которых еда, алкоголь или вечернее переедание связаны со стрессом и усталостью;
- люди, которым нужен не мотиватор, а понятная система.

## Основные пользовательские сценарии

### 1. Диагностика

Пользователь фиксирует исходную точку:

- вес;
- талия;
- сон;
- энергия;
- питание;
- движение;
- алкоголь;
- главная проблема;
- желаемое состояние;
- текущий уровень развития.

### 2. Ежедневный чек-ин

Пользователь каждый день отвечает на короткий набор вопросов:

- вес утром;
- сон;
- энергия утром / днём / вечером;
- движение;
- питание;
- алкоголь;
- пищевой импульс;
- переедание;
- главный триггер дня;
- один вывод;
- следующий ход.

### 3. Карта разности потенциалов

Приложение помогает сформулировать:

- что есть сейчас;
- что возможно;
- что между ними создаёт напряжение;
- какая внутренняя задача возникает;
- какой ближайший ход реален.

### 4. Выбор главного механизма

После 3 дней наблюдения пользователь выбирает один механизм:

- вечернее переедание;
- алкоголь;
- недосып;
- отсутствие движения;
- хаотичное питание;
- сладкое после стресса;
- еда от усталости;
- еда от скуки;
- отсутствие заранее приготовленной еды.

### 5. Практика недели

Пользователь работает не со всем сразу, а с одним механизмом.

Пример:

> “7 дней я не исправляю всю жизнь. Я тренирую один переход: вечернее напряжение → не еда автоматически, а пауза, вода, 10 минут ходьбы и нормальный ужин.”

### 6. Обратная связь

Приложение не хвалит и не ругает, а показывает:

- что повторяется;
- где есть улучшение;
- какой механизм влияет сильнее всего;
- какая новая внутренняя задача появилась.

## Принцип тона

Тон приложения:

- спокойный;
- точный;
- без мотивационного шума;
- без стыда;
- без “возьми себя в руки”;
- без эзотерики;
- с акцентом на факт, действие и обратную связь.

## Главное отличие от диетических приложений

Обычное приложение спрашивает:

> “Сколько ты съел?”

Selection Point спрашивает:

> “Что произошло до того, как ты съел? Какой механизм включился? Какую внутреннюю задачу это показывает?”

## MVP success criteria

Первая версия считается успешной, если пользователь за 14 дней:

1. Заполнил диагностику.
2. Сделал минимум 10 ежедневных чек-инов.
3. Назвал 1–3 повторяющихся механизма.
4. Сформулировал одну внутреннюю задачу.
5. Прошёл минимум один 7-дневный цикл работы с механизмом.
6. Получил понятную обратную связь по весу, энергии, сну, пищевым импульсам и поведению.
