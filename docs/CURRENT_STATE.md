# CURRENT_STATE

**Статус:** совместимая точка входа.  
**Авторитетный динамический документ:** `docs/FOUNDATION/CURRENT_PROJECT_STATE.md`.

Этот файл больше не ведётся как самостоятельный источник истины. Он существует для совместимости со старыми ссылками и направляет читателя к актуальной архитектуре.

## Актуальная контрольная точка

- Активная фаза: **Фаза 3 — точная архитектура пяти ступеней**.
- Активная ступень: **Ступень 1 «Жизнь внутри автоматизма»**.
- Утверждены параметры 1–4:
  1. состояние человека;
  2. способ восприятия реальности;
  3. способ восприятия себя;
  4. что управляет решениями.
- Следующий вопрос: **параметр 5 — доступная точка выбора**.

## Введённая архитектура знаний

Для контроля растущего теоретического объёма проект использует четыре слоя:

```text
полная теория
→ каноническая архитектура
→ общие библиотеки
→ материалы доставки
```

Созданы и действуют:

- `docs/FOUNDATION/CANONICAL/00_CANONICAL_RULES.md`;
- `docs/FOUNDATION/CANONICAL/01_HUMAN_CHANGE_MODEL_CANONICAL.md`;
- `docs/FOUNDATION/CANONICAL/02_STAGE_1_CANONICAL.md`;
- `docs/FOUNDATION/LIBRARIES/00_INDEX.md`;
- реестры терминов, утверждений, примеров, рисков, доказательного статуса и связей.

Параметры 1–4 первой ступени канонизированы. Полные теоретические файлы не сокращены и остаются основаниями утверждённых решений.

Начиная с параметра 5 после каждого утверждения обязательно создаются полный файл, каноническая карточка и обновления библиотек. Новый полный файл применяет правило дельты и подробно раскрывает только новый вопрос.

## Принцип приоритета

При любом расхождении между этим файлом, ранними продуктовыми документами и материалами в `docs/FOUNDATION/` приоритет имеют утверждённые документы `docs/FOUNDATION/` и полные теоретические источники.

Канонический слой не может незаметно изменить утверждённый смысл. При расхождении с полным источником работа останавливается до разрешения противоречия.

## Что сейчас запрещено считать утверждённым

До соответствующих фаз нельзя считать принятыми решениями:

- 14-дневный протокол;
- фиксированное количество уроков;
- готовую карту модулей курса;
- mobile-first формат;
- приложение;
- AI-сопровождение;
- Expo, React Native или иной стек;
- backend, хранилище, авторизацию и платежи.

## Следующее действие

Продолжать работу только с SP-S1-P05 — «доступная точка выбора» — без смешения с главным навыком, главной иллюзией, главной болью, практиками или критериями перехода.

При подготовке параметра использовать:

- SP-HCM-01–SP-HCM-08;
- SP-S1-P01–SP-S1-P04;
- правило дельты;
- общие библиотеки вместо повторного полного пересказа предыдущих параметров.
